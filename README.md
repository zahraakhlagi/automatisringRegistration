# automatisringRegistration

Continuous Integration (CI) är ett arbetssätt inom mjukvaruutveckling där utvecklare regelbundet lägger in sin kod i ett gemensamt kodförråd, till exempel GitHub. Varje gång ny kod skickas in körs automatiska kontroller, som att bygga programmet och köra tester. Syftet med CI är att upptäcka fel så tidigt som möjligt, istället för att hitta problem sent i projektet. På så sätt hålls koden i ett fungerande skick hela tiden.
En stor styrka med Continuous Integration är att fel upptäcks snabbt. Utvecklare får direkt feedback om något inte fungerar, vilket gör det lättare att rätta till problem. CI bidrar också till bättre kodkvalitet och minskar risken för stora konflikter när flera personer arbetar med samma kod. En svaghet med CI är att det kräver att automatiska tester finns på plats. Det tar också tid i början att sätta upp CI och skriva bra tester. Om testerna är dåligt skrivna kan de dessutom ge falska felmeddelanden.
Testautomatisering innebär att tester skrivs som kod och körs automatiskt utan att en person behöver testa manuellt. Exempel på detta är enhetstester med JUnit eller användargränssnittstester med Selenium. Testautomatisering sparar tid, minskar risken för mänskliga fel och gör det möjligt att köra tester ofta. Nackdelen är att det tar tid att skriva tester och att de måste uppdateras när systemet förändras. Alla delar av ett system går heller inte att automatisera.
Testautomatisering är starkt kopplat till CI och CI/CD. CI använder automatiska tester för att kontrollera att ny kod fungerar korrekt. Om ett test misslyckas stoppas processen och koden godkänns inte. CI/CD bygger vidare på detta genom att även kunna leverera eller publicera koden automatiskt när alla tester är godkända. Utan testautomatisering skulle CI och CI/CD inte fungera på ett effektivt sätt.
För att tester ska köras automatiskt i GitHub används GitHub Actions. Först måste projektet och testerna finnas i ett GitHub-repository. Därefter skapas ett workflow, vilket är en YAML-fil som placeras i mappen .github/workflows. I denna fil beskrivs när tester ska köras, till exempel vid varje push, och vilka steg som ska utföras. När koden pushas till GitHub startar GitHub Actions automatiskt. Miljön sätts upp, till exempel genom att installera Java och nödvändiga verktyg, och sedan körs testerna automatiskt. Resultatet visas direkt i GitHub, där man kan se om testerna har passerat eller misslyckats.
Sammanfattningsvis handlar Continuous Integration om att ofta integrera kod och testa den automatiskt. Testautomatisering är grunden som gör detta möjligt, och tillsammans med CI/CD skapas ett stabilt och effektivt sätt att utveckla och leverera programvara med hög kvalitet.
